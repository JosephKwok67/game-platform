# 更新说明：贪吃蛇排行榜升级 + 保存体验修复

> 日期：2026-07-01
> 相关 PR：[#1](https://github.com/JosephKwok67/game-platform/pull/1)

## 一、问题背景

用户反馈「游戏数据保存不了」，并希望在贪吃蛇里加一个数据排行榜。

## 二、诊断结论

后端其实工作正常：

- `scores` 表存在，且已有真实记录（例如玩家「god」有一条 410 分记录）。
- `profiles` / `rooms` / `friendships` 等表均存在，且通过 RLS 可正常公开读取。

**真正原因**：用户以「访客」身份游玩。未登录时，分数只写入浏览器 `localStorage`，**不会上传到数据库**，因此看起来像"保存不了"。只有登录后，`saveScore` 服务端动作才会把成绩写入 `scores` 表。

**次要隐患**：原 schema 没有"注册时自动建 profile"的数据库触发器，profile 依赖前端 `AuthButton.tsx` 手动 insert。一旦该 insert 失败（网络 / 时序 / RLS），排行榜就会显示"匿名"。

## 三、本次改动

均为前端增量 + 附加式 SQL，安全、无破坏性。

### 1. `components/SnakeLeaderboard.tsx` — 排行榜组件升级
- 模式切换 Tab：全部 / 无尽 / 闯关
- 按用户去重，只取每人最高分
- 显示分数、模式标签、日期
- 高亮当前玩家（青色 + "(我)"）
- 玩家排名在榜单之外时，单独显示"我的排名"

### 2. `components/games/SnakeGame.tsx` — 保存体验修复
- 未登录时，游戏页顶部显示醒目橙色横幅："⚠️ 你正在以「访客」身份游戏，分数不会上传…去登录 →"
- 新增常驻「🏆 排行榜」按钮 + 弹窗，游戏中随时查看
- 游戏结束页高亮本人成绩，并引导登录上榜

### 3. `app/page.tsx` — 大厅首页登录引导
- 未登录：橙色提示"登录后游戏成绩才会上传到全球排行榜，否则只保存在本机"
- 已登录：绿色确认"已登录，成绩会自动上传到全球排行榜"

### 4. `database/migrations/002_auto_profile.sql` — 注册自动建 profile（数据库）
- 幂等迁移脚本
- 为历史遗漏用户补建 profile
- 新增 `handle_new_user()` 触发器函数（SECURITY DEFINER）
- 绑定 `on_auth_user_created` 触发器到 `auth.users`

## 四、需要手动执行的一步 ⚠️

第 4 项数据库触发器无法远程执行，需在 Supabase 控制台手动跑一次：

1. 打开 Supabase 控制台 → 你的项目 → 左侧 **SQL Editor**
2. 打开仓库文件 `database/migrations/002_auto_profile.sql`，整段内容粘贴进去
3. 点 **Run**（脚本幂等，可安全重复执行）

执行后，新注册用户会自动生成用户名，排行榜不再出现"匿名"。

## 五、部署与验证

- 所有前端改动已合并到 `main`，Vercel 自动部署到 https://game-platform-psi.vercel.app
- 已用真实浏览器验证：访客横幅、排行榜按钮、模式 Tab、游戏结束页、首页登录引导条均正常渲染。

## 六、常见疑问

**Q：为什么我以前的分数"没保存"？**
A：因为你在以访客身份玩。登录后重新游玩，成绩就会自动上榜。已登录用户的成绩一直是正常保存的。

**Q：排行榜为什么显示"匿名"？**
A：个别用户的 profile 创建失败导致。执行第四步的 SQL 迁移后即可修复，且新用户不再出现该问题。
